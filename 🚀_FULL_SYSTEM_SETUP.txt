🎯 QUICK SETUP (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════

TERMINAL 1: BACKEND
────────────────────────────────────────────────────────────────────────────

1. Open terminal in project root

2. Install Python dependencies:
   pip install -r requirements.txt

3. Run backend:
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

✓ Backend running on http://localhost:8000
✓ API docs on http://localhost:8000/api/docs


TERMINAL 2: FRONTEND
────────────────────────────────────────────────────────────────────────────

1. Open new terminal in project root

2. Navigate to frontend:
   cd frontend

3. Install dependencies:
   npm install

4. Run frontend:
   npm run dev

✓ Frontend running on http://localhost:3000
✓ Browser opens automatically

🔄 DATA FLOW
═══════════════════════════════════════════════════════════════════════════

1. USER SIGNS UP
   Frontend → POST /auth/signup → Backend
   Backend → Create user → Database
   Backend → Return token → Frontend

2. USER VIEWS EVENTS
   Frontend → GET /events → Backend
   Backend → Query database → Return events
   Frontend → Display events with animations

3. USER SELECTS SEATS
   Frontend → POST /bookings/lock-seats → Backend
   Backend → Lock seats (5 min) → Database
   Backend → Return lock_id → Frontend
   Frontend → Show selected seats with glow

4. USER CONFIRMS BOOKING
   Frontend → POST /bookings/confirm → Backend
   Backend → Verify lock → Create booking → Database
   Backend → Return confirmation → Frontend
   Frontend → Show confetti animation

🔧 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

FRONTEND ISSUES
├─ Animations not showing → Hard refresh (Ctrl+Shift+R)
├─ Styles broken → Delete node_modules, reinstall
├─ Port 3000 in use → npm run dev -- --port 3001
└─ Slow animations → Close other tabs, refresh

BACKEND ISSUES
├─ Module not found → pip install -r requirements.txt
├─ Port 8000 in use → python -m uvicorn app.main:app --port 8001
├─ Database error → Check .env file
└─ JWT error → Check SECRET_KEY in .env
